# ScratchMath Graph Feedback Generation

Use this skill as the final per-sample feedback step after OCR, student work extraction, reference solving, and scratchwork diagnosis have completed.

## Runtime Role

This skill is implemented by `skills/feedback_generation.py` and called by `ScratchMathFeedbackAgent` as round 5. It first builds an `analysis_graph`, then asks the model to reason over that graph before producing feedback.

## Input

- `sample`: normalized question and image metadata.
- `parse`: OCR parse record.
- `diagnosis`: diagnosis record.
- `student_work`: extracted student answer and steps.
- `reference_solution`: independent reference answer and steps.
- `analysis_graph`: optional prebuilt graph.

## Output

- `student_feedback`: student-facing explanation.
- `next_step_advice`: targeted next practice advice.
- `teacher_summary`: teacher-facing summary.
- `raw.graph_reasoning_summary`: model's graph-based reasoning summary.
- `raw.analysis_graph`: nodes, edges, and graph-level summary used as evidence.

## Operational Notes

- The graph contains question, OCR, student answer, student steps, uncertainties, reference answer, reference steps, concepts, diagnosis, and evidence nodes.
- This is the per-sample graph reasoning skill. Class-level social graph reporting is generated by `scripts/build_student_social_report.py`.
- When previous rounds already exist, rerun only this skill with `scripts/regenerate_graph_feedback.py`.
